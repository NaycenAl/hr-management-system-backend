## Documentation

The backend documentation follows the **Diátaxis framework**, which structures documentation into four types:

1. **Tutorials** – Step-by-step guides to help new developers get started.
2. **How-to guides** – Practical instructions to perform specific tasks (e.g., adding an employee, requesting a leave).
3. **Reference** – Detailed technical documentation of the API endpoints, models, serializers, and services.
4. **Explanation** – Conceptual documentation explaining architecture choices (e.g., Hexagonal architecture, TDD workflow) and the reasoning behind design decisions.

The documentation is located in the `docs/` folder and can be viewed locally or deployed as a website using MkDocs or Sphinx.
